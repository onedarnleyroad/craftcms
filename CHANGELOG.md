# onedarnleyroad/craftcms Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

### [Unreleased]

### Added

- `img.responsive()`, `img.dummy()` and `svg.transparent()` macros

### Changed

- Remove Imager-X

### Fixed:

- Fix 500 and 503 pages

## [4.0.0]

### Added:

- Now built for Craft 5

### Changed:

- CKEditor replacing Redactor plugin
- DDEV now uses PHP 8.2 and MySQL 8.0
- Now uses Vite 5 and Node 20
- Default Homepage amends

## [3.0.1]

### Added:

- DDEV config specifies that the project is of type `craftcms`

## [3.0.0]

### Added:

- Now built for Craft4

### Changed:

- set preview token duration to 1 week
- Run `craft up` _first_ when running composer install/update
- increase max revisions to 10
- Default to Servd as hosting environment (remove Async Queue and adjust `make pull` command)
- Switch to Verbb's Hyper as the default plugin for link fields
- Remove make commands that can now be fulfilled with `ddev`
- Use `expose` instead of `port` for Vite

### Fixed:

- Fix Vite restart when contents in `./templates` change (https://github.com/onedarnleyroad/craftcms/pull/38)
- Avoid reloading when JS changes (thanks @nystudio107)

## [2.3.1]

### Added:

- Add `vbenjs/vite-plugin-compression` support
- Add sourcemaps to production build

### Fixed:

- Fix `ElMassimo/vite-plugin-manifest-sri` support

## [2.3.0]

### Added:

- Add a boilerplate Site Module
- Ensure `base.twig` does not introduce whitespace at start of response
- New `HARD_MODE` to throw deprecation errors by environment

### Changed:

- Updated for DDEV 1.19 compatibility
- Default to Node version 16, composer 2
- Update Craft, plugins & NPM packages to latest
- Upgrade Tailwind 3.x
